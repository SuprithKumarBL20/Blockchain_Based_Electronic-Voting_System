BLOCKCHAIN WORKFLOW:
===================

1. VOTE CREATION:
   - Voter selects candidate
   - System creates vote_hash = SHA256(voter_id:candidate_id:timestamp)
   - Voter hash created = SHA256(voter_email:voter_phone)
   - Vote recorded in database with pending status

2. BLOCK CREATION:
   - New Block object created with:
     * index: next block number
     * timestamp: current datetime
     * voter_hash: anonymized voter identifier
     * vote_hash: encrypted vote data
     * previous_hash: hash of previous block

3. MINING PROCESS:
   - Block mining starts with nonce = 0
   - Target difficulty: 4 leading zeros (configurable)
   - Hash recalculated with incremented nonce
   - Mining continues until hash meets difficulty requirement
   - Average mining time: ~1-2 seconds per vote

4. BLOCKCHAIN STORAGE:
   - Mined block added to blockchain chain
   - Block data saved to BlockchainBlock table
   - Vote status updated to 'confirmed'
   - blockchain_hash field updated with block hash

5. VERIFICATION:
   - Any block can be verified by recalculating hash
   - Chain integrity verified by checking previous_hash links
   - Vote authenticity verified through vote_hash

VOTE ENCRYPTION PROCESS:
=======================

1. VOTER IDENTIFICATION HASH:
   Input: voter_email + voter_phone
   Process: SHA256(email:phone)
   Output: 64-character hexadecimal hash
   Purpose: Anonymous voter identification

2. VOTE DATA HASH:
   Input: voter_id + candidate_id + timestamp
   Process: SHA256(voter_id:candidate_id:timestamp)
   Output: 64-character hexadecimal hash
   Purpose: Tamper-proof vote recording

3. BLOCK HASH:
   Input: Block data (index, timestamp, voter_hash, vote_hash, previous_hash, nonce)
   Process: SHA256(JSON.stringify(block_data))
   Output: 64-character hexadecimal hash
   Purpose: Block integrity and mining

SECURITY MEASURES:
=================

1. MULTI-FACTOR AUTHENTICATION:
   - Password authentication (bcrypt hashing)
   - Face recognition (ONNX model, 60% threshold)
   - OTP verification (6-digit, 5-minute expiry)

2. DATA PROTECTION:
   - Passwords: Bcrypt with 12 rounds of salting
   - Votes: SHA256 hashing with voter anonymization
   - Personal data: Face encodings stored securely

3. NETWORK SECURITY:
   - Rate limiting: 200 requests/day, 50/hour per IP
   - JWT tokens for API authentication
   - CORS protection enabled
   - Input validation on all endpoints

4. BLOCKCHAIN SECURITY:
   - Proof-of-Work prevents spam
   - Immutable ledger prevents vote tampering
   - Cryptographic hashing ensures integrity
   - Previous hash linking prevents chain modification

FACE RECOGNITION PROCESS:
========================

1. MODEL LOADING:
   - ONNX model loaded at application startup
   - Model path: ai/model.onnx
   - Cross-platform compatibility ensured
   - Memory optimization implemented

2. FACE CAPTURE:
   - Webcam access through browser API
   - Canvas element for photo capture
   - Image preprocessing (resize, normalize)
   - Quality validation (face detection)

3. ENCODING GENERATION:
   - Face landmarks extracted
   - 128-dimensional encoding created
   - Normalization applied
   - Encoding stored as base64 string

4. SIMILARITY COMPARISON:
   - Cosine similarity calculation
   - Threshold: 0.6 (60% similarity required)
   - Multiple attempts allowed
   - Fallback to OTP if face fails
